## ğŸ’¡ Quick Reference for Presentation

**When asked about OOP:**

1. **Encapsulation:** "We use private fields with public getters/setters to protect data"
2. **Inheritance:** "SavingsAccount and CurrentAccount inherit from Account class"
3. **Polymorphism:** "Same method calculateInterest() behaves differently in each account type"
4. **Abstraction:** "Abstract Account class hides implementation details"
5. **Composition:** "Account contains a list of Transactions"

**Example to demonstrate:**
```java
// Show inheritance and polymorphism
Account savings = new SavingsAccount("ACC001", "John", 10000, 1);
Account current = new CurrentAccount("ACC002", "Jane", 5000, 2);

savings.calculateInterest();  // Returns 4% interest
current.calculateInterest();  // Returns 2% interest

// Same method, different behavior! That's polymorphism!
```
